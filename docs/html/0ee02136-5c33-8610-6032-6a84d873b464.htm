      BOOSEapp Namespace

      BOOSE application: contains application-specific command implementations, canvas helpers and the Windows Forms UI used for the sample BOOSE application.

      Classes

      AppArray Command implementation for creating arrays in the BOOSE variable store. Allocates an array with a specified name and size for use in BOOSE programs.
      AppAssign Command implementation for variable assignment in BOOSE. Evaluates expressions and assigns values to variables in the variable store.
      AppCall Command implementation for calling user-defined methods in BOOSE. Supports fuzzy matching of method names and evaluates arguments at runtime.
      AppCanvas A simple canvas implementation that draws onto an in-memory Bitmap. Implements the Canvas contract from the BOOSE library.
      AppCircle Command implementation for Circle which draws a circle centred at current pen position.
      AppClear Command to clear the canvas.
      AppCommandFactory Factory that produces application-specific command implementations for the BOOSE parser.
      AppDrawTo Command implementation for DrawTo which draws a line from the current pen position to the given coordinates.
      AppEndForMarker Marker command representing the end of a FOR loop.
      AppEndMarker Marker command representing the 'end if' keyword. Pops the IF execution frame from the execution context stack.
      AppEndMethodMarker Marker command representing the end of a method definition.
      AppEndWhileMarker Marker command representing the end of a WHILE loop.
      AppFor Lightweight marker command for the start of a FOR loop in BOOSE. Actual execution logic is handled by AppForComposite.
      AppForComposite Composite command that handles the actual execution logic of a FOR loop, including variable initialization, step increments, and body iteration.
      AppIf Command implementation for conditional IF statements in BOOSE. Evaluates a conditional expression and manages execution context for THEN/ELSE branches. Supports comparison operators (==, !=, <, >, <=, >=) and boolean expressions.
      AppIfComposite Composite IF command created after parsing to group THEN/ELSE children. Evaluates condition once and executes the appropriate branch.
      AppIfMarker Marker command used to separate THEN and ELSE branches in IF statements. Switches execution context to the ELSE branch when encountered.
      AppInt Command implementation for integer variable declarations in BOOSE. Declares and optionally initializes integer variables. Values are stored as doubles in the variable store.
      AppMethod Lightweight marker command for the start of a METHOD definition in BOOSE. Actual registration logic is handled by AppMethodComposite.
      AppMethodComposite Composite command that handles method registration and invocation, including parameter handling and return value management.
      AppMethodRegistry Registry for user-defined methods in the BOOSE application, allowing for persistent method storage and lookup.
      AppMoveTo Command implementation for the MoveTo command which moves the pen to the specified coordinates without drawing.
      AppPeek Command implementation for retrieving array element values (peek operation). Reads a value from an array at a specified index and stores it in a variable. Supports fuzzy array name matching (plural/singular) and runtime index evaluation.
      AppPen Command implementation for Pen which sets the drawing colour using RGB integer components.
      AppPoke Command implementation for setting array element values (poke operation). Assigns a value to an array at a specified index. Automatically creates arrays if they don't exist. Supports runtime expression evaluation for both index and value.
      AppReal Command implementation for real (floating-point) variable declarations in BOOSE. Declares and optionally initializes real number variables in the variable store.
      AppRect Command implementation for Rect which draws a rectangle at the current pen position.
      AppSet Command implementation for the 'set' variable assignment command in BOOSE. Alternative syntax for variable assignment using 'set name = expr' format.
      AppSettings Static class containing global application settings for the BOOSE application.
      AppVariableStore Singleton store for managing variables (numbers, arrays, and booleans) in the BOOSE application. Provides methods for variable declaration, retrieval, and expression evaluation.
      AppWhile Lightweight marker command for the start of a WHILE loop in BOOSE. Actual execution logic is handled by AppWhileComposite.
      AppWhileComposite Composite command that handles the actual execution logic of a WHILE loop, including condition evaluation and body iteration.
      AppWhileMarker Marker command used by the parser to identify blocks within a WHILE loop.
      AppWrite Command implementation for writing text to the canvas. Supports string literals, variable evaluation, expression concatenation, and optional coordinate positioning. Handles currency symbols and can automatically format numeric outputs.
      DebugLogger Simple logging helper used by application commands to emit debug information to a configurable sink.
      ExecutionContext Coordinates the execution flow of BOOSE commands, managing a stack of frames to handle conditional execution (IF/ELSE blocks).
      Form1 Main form for the BOOSE application. ///

      Interfaces

      IHasHeader Interface for commands that expose a header string (e.g., method or for loop headers). Provides a default implementation that returns the Header property (never null).
